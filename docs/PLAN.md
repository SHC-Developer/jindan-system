# PLAN.md

## 프로젝트 개요

Slack 시스템 장점을 결합한 **안전진단 업무 자동화 시스템** 웹앱.  
반응형 웹으로 제작, TypeScript 사용, Firebase 백엔드, 향후 모바일 앱 확장 가능.

- **목표**: 대메뉴/중메뉴/채팅 탭 구조로 프로젝트별 협업·자동화 기능 제공
- **대상 사용자**: 관리자 2명, 일반 5명 (MVP, 회원가입 없음)

---

## MVP 스코프 (1차)

1. **로그인**
   - 관리자 2명 / 일반 5명
   - 회원가입 없음, 계정은 Firebase에 직접 입력 후 배포

2. **채팅 기능**
   - 대메뉴 / 중메뉴 / 채팅 탭 구조
   - 프로젝트·기능 단위로 채널 구분

3. **5개 자동화 기능 기본 뼈대**
   - 물량표, 사진첩 등 5개 기능의 기본 UI/라우팅만 구성

---

## 개발 단계

| 단계 | 내용 | 비고 |
|------|------|------|
| 1 | **로그인** | Firebase Auth, 역할(관리자/일반) 구분, 보호된 라우트 |
| 2 | **채팅** | 대메뉴·중메뉴·채팅 탭 구조, Firestore 메시지/파일 |
| 3 | **자동화 1** | 첫 번째 자동화 기능 뼈대 (예: 물량표) |
| 4 | **자동화 2** | 두 번째 자동화 기능 뼈대 |
| 5 | **자동화 3** | 세 번째 자동화 기능 뼈대 |
| 6 | **자동화 4** | 네 번째 자동화 기능 뼈대 |
| 7 | **자동화 5** | 다섯 번째 자동화 기능 뼈대 |

진행 순서: **로그인 → 채팅 → 자동화 1개씩**.
